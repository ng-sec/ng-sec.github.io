凡事预则立，不预则废

3.1 规划前思考

但你觉得辛苦的时候问自己三个问题：

  • 你为什么会辛苦？ 发现你想做的事情下属没帮你去做
  • 你为什么很辛苦？ 你发现每一个小时的效率不够
  • 你为什么那么辛苦？ 你发现很多人做的事情并不真正产生效益

金融企业战略规划（通常是5年）,IT战略规划（通常是3年），信息安全三年规划、XX年工作计划，是自上而下、一脉相承的。负责制定IT战略规划的人，通常要求企业安全负责人提供信息安全三年规划作为基础材料，统筹而成。一份看起来高大上，实施起来又接地气可执行的信息安全规划，是金融企业安全负责人的必备技能。

信息安全规划，决定了新一年的安全投入，包括人员和资金的预算。

信息安全规划编制启动时间点选择方面，建议选择每年10月启动，12月定稿。

3.2 规划框架

安全规划框架应该包括：概述、需求分析和安全目标、各个安全领域的现状和差距分析、解决方案和计划、安全资源规划、当年重点项目和重点任务、上一版安全规划目标差距分析等。

  • 概述：主要包括信息安全形势分析，包括外部安全形势、行业形势、监管和股东要求、对手分析

  • 安全目标：规划周期内应该达到什么样的信息安全水平。安全目标应该是我们花费大力气才能达到的。

  • 现状和差距分析：相当于自我体验，最重要的是能从过往的安全检查中分析管理差距，以及白盒检测和黑盒检测（众测和红蓝对抗）中获得技术差距。差距分析的重点是能够及时发现自己的漏洞和弱点，对这类检测失效的原因进行深挖。

  • 解决方案和计划，解决方案是提出的一条条解决措施，最终要落到重点项目和任务上去实现。

  • 当前的重点项目和重点任务，是解决方案落地的关键，当年的工作目标靠重点项目和重点任务实现。

  • 上一版安全规划目标差距分析，是针对上一阶段的规划执行情况进行回顾和检讨，主要是通过历史经验中吸取教训，然后不断的改进。
